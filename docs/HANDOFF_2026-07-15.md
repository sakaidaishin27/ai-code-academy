## 1. このセッションで何を直したか（要点）

### A. 前提をデスクトップアプリに全面変更（commit 55c675c ほか）
- Claude Code は**デスクトップアプリ（Claude Desktop・Mac）前提**。学習者はターミナルを触らない＝操作は「アプリのボタン・ブラウザ・日本語」だけ。新しい会話＝「+ 新しいセッション」（Cmd+N）。

- ターミナルの言及は第0章 l0-3 の脚注1箇所だけ（意図的に残置）。

### B. 第4章 Google接続を作り直した（commit 937f7b1）— L1の到達点が壊れていた
実機検証で判明した事実（詳細は `docs/factcheck.md` の gog 節）:
- インストールは `brew install openclaw/tap/gogcli`（**`brew install gog` は存在しないパッケージ**）。コマンド名 gog / パッケージ名 gogcli。
- gogは**自分のGoogle CloudでOAuthクライアント（Desktop app）を自作するのが必須**。この工程を省くと認証時にブラウザすら開かない。
- **Testing のままだと7日で失効**する → Cloud Console の Audience で「アプリを公開」まででワンセット。
- 未読メールは **`gmail search "is:unread" --max 5`**（`gmail list --unread --limit 5` は実在しないフラグ）。
- 安全設計: `auth add <email> --services calendar,gmail --readonly` で読み取り専用に絞り、「メール送信は人間の役目」を仕組みで担保。
→ 第4章を3実習に再構成（l4-2 道具を入れる／l4-3 Googleの鍵を作って繋ぐ／l4-4 朝の一言）。Cloud Console 5画面は直リンクで提示。

### C. 第8章 GitHub認証（commit 937f7b1）
- 素の `gh auth login` は矢印キーTUIでAIが操作できない → `gh auth login --web --git-protocol https --hostname github.com --skip-ssh-key` を**バックグラウンドで**実行し、8桁のワンタイムコードをユーザーに見せてブラウザ承認（実機で挙動確認済み）。

### D. 関門(verify)の穴を全て塞いだ（commit 937f7b1 + 40bdeed）

- `tools/gate-attack-test.mjs` を新設し全数テスト（現在PASS）。**関門を足したらこのテストにケースも足す**（未テストは警告が出る）。
